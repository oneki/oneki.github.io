Oneki.js hooks

Oneki.js provides several hooks to speed up the developpement and to hide most of the complexity of Redux, Redux Saga and Immer.

Configuration hooks#

HookDescription
useSettingsuseSettings returns the whole object passed as a prop to App (generally defined in settings.js)
useSettinguseSetting returns a specific entry from Settings

AJAX hooks#

HookDescription
useGetuseGet sends an ajax GET request and returns back the data.
useSecureGetuseSecureGet is similar to useGet but adds a Bearer authorization header that contains the token received and stored by useLoginService in the redux store
usePostusePost sends an ajax POST request and returns back the payload received from the server
useSecurePostuseSecurePost is similar to usePost but adds a Bearer authorization header that contains the token received and stored by useLoginService in the redux store
usePutusePut sends an ajax PUT request and returns back the payload received from the server
useSecurePutuseSecurePut is similar to usePut but adds a Bearer authorization header that contains the token received and stored by useLoginService in the redux store
useDeleteuseDelete sends an ajax DELETE request
useSecureDeleteuseSecureDelete is similar to useDelete but adds a Bearer authorization header that contains the token received and stored by useLoginService in the redux store

Notification hooks#

HookDescription
useNotificationServiceuseNotificationService provides a central service to send notification.
useNotificationsuseNotifications listens on a topic that contains notifications.
As soon as the topic is updated, the component does a re-render

Service hooks#

HookDescription
useLocalServiceThe goal of useLocalService is to create a service local to the component with two kinds of methods:
  • Reducer methods: the role of these methods is to update the local state of the component. These methods are generally very simple.
  • Saga methods:
    • These methods are generally asynchronous and more complex
    • A saga method is more like a flow. Each step of the flow can be asynchronous and can trigger a re-render
    • A saga method never updates the local state by itself. It always calls a Reducer method to do that.
useReduxServiceThe goal of useReduxService is to create a singleton with two kinds of methods:
  • Reducer methods: the role of these methods is to update the redux state. These methods are generally very simple.
  • Saga methods
    • These methods are generally asynchronous and more complex.
    • A saga method is more like a flow. Each step of the flow can be asynchronous and can trigger a re-render
    • A saga method never updates the Redux state by itself. It always calls a Reducer method to do that.
useReduxSelectoruseReduxSelector is similar to useSelector from React Redux but is tailored to be used with a Redux Store built by Oneki.js
